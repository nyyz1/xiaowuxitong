# 学校试点电脑部署清单

## 适用场景

这份清单用于项目从“本地演示”推进到“学校试点电脑小范围试点”时的协调和落地。

建议配合阅读：

- `docs/approval-brief.md`
- `docs/security-and-deployment-plan.md`
- `docs/postgresql-install-and-acceptance-runbook.md`
- `docs/deployment-and-smoke-test.md`

## 目标

在不影响学校现有 Excel 流程的前提下，完成一次可控、可回退、可验收的高二 PostgreSQL 试点部署。

## 参与角色

- 项目负责人：负责推进、沟通、组织验收
- 学校网管/信息中心：负责服务器、数据库、网络与备份
- 试点操作老师：负责导入、查询、导出、检查录入试跑
- 审批人：确认服务器可用、风险可控、试点范围明确

## 一、试点前先确认的 8 件事

- [ ] 已明确试点目标：先验证登录、导入、筛选、导出、检查录入和统计
- [ ] 已明确试点范围：当前先在高二年级试用，或使用脱敏样例数据
- [ ] 已明确试点电脑位置：当前这台安装 PostgreSQL 的办公电脑
- [ ] 已明确网络方式：哪些办公电脑可以访问，是否限制为校内网
- [ ] 已明确谁负责安装 PostgreSQL、谁负责安装 Node.js、谁负责后续巡检
- [ ] 已明确是否允许 Docker；若不允许，则采用原生 PostgreSQL 安装
- [ ] 已明确备份位置、备份负责人和恢复演练责任人
- [ ] 已明确试点期间保留原有 Excel 流程作为兜底

## 二、建议的服务器最低准备

- [ ] 试点电脑具备管理员权限，可安装 PostgreSQL 和 Node.js
- [ ] 试点电脑可用操作系统：Windows 优先，后续如迁移再考虑服务器
- [ ] 可用 CPU 至少 2 核
- [ ] 可用内存至少 4 GB
- [ ] 可用磁盘至少 20 GB，并在 `D` 盘预留数据库与备份目录
- [ ] 试点电脑时间同步正常
- [ ] 试点电脑可访问项目文件，或允许把部署包拷入本机

## 三、需要学校网管提供的信息

- [ ] 试点电脑名称、IP、操作系统版本
- [ ] PostgreSQL 本机监听端口
- [ ] 数据目录位置（建议 `D:\PostgreSQL\data`）
- [ ] 备份目录位置
- [ ] 允许访问系统的客户端网段或办公电脑范围
- [ ] 数据库管理员账号，或代为执行建库建用户操作
- [ ] 应用运行账号，或是否允许先用当前登录用户试运行

## 四、试点当天建议执行顺序

1. 安装并启动 PostgreSQL
2. 创建试点数据库和应用数据库用户
3. 配置 `.env.local`，确认 `DATABASE_URL` 可连接
4. 执行 `npm install`、`npm run db:generate`、`npm run db:push`
5. 执行 `npm run db:seed:demo` 写入演示数据
6. 启动应用，优先用 `start-school-pilot.cmd` 或 `npm run pilot:school` 验证
7. 按 `docs/deployment-and-smoke-test.md` 完成浏览器烟测
8. 导出一份数据库备份并记录备份位置
9. 记录试点问题、账号信息、服务器信息和下一步安排

## 五、试点验收通过标准

- [ ] 数据库连接稳定，应用可正常启动
- [ ] `admin`、`leader1~leader3`、`grade11.manager1~grade11.manager3`、`data.manager`、`inspector` 演示账号能登录
- [ ] 校领导账号可查看全校数据并执行校级导出，但不能进入系统管理员页面
- [ ] 校领导账号可进入 `/dashboard/inspection` 查看全校检查记录，但不能录入或修改
- [ ] 年级管理员账号只能访问绑定年级的学生、检查和统计数据，不能越权查看其他年级
- [ ] `/dashboard/users` 可新增、停用、启用、重置密码
- [ ] `/dashboard/structure` 可新增、重命名、删除临时班级
- [ ] `/dashboard/people` 可筛选、下载模板、导出教师/学生，并可由有权限账号修改或删除误录档案
- [ ] `/dashboard/inspection` 可录入和查询检查记录
- [ ] `/dashboard/exports` 可导出 Excel/CSV 检查统计报表
- [ ] `AuditLog` 有新增记录，能证明导出和用户管理动作被留痕
- [ ] 完成至少一次数据库备份

## 六、试点后要留存的材料

- [ ] 试点电脑 IP、部署路径、数据库名、数据库用户名
- [ ] 项目 `.env.local` 的安全保存版本
- [ ] 试点当天执行过的命令记录
- [ ] 浏览器烟测结果
- [ ] 备份文件位置和恢复负责人
- [ ] 已知问题清单和下一次处理计划

## 七、如果当天条件不足，优先级怎么排

1. 先保证 PostgreSQL 能装好并能在本机 `localhost` 连通
2. 再保证应用能启动并能让同一校内网办公电脑访问
3. 再写入演示数据并完成浏览器烟测
4. 最后再讨论高二真实数据试点

## 八、明确不在本次试点里解决的事

- 不在试点第一天导入全校真实数据
- 不在试点第一天替代原有 Excel 流程
- 不在试点第一天扩展到成绩、排课、家校沟通等新模块
- 不在没有备份方案前进入正式投产
