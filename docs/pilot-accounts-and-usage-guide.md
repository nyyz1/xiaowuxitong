## 四、日常启动方法

推荐直接双击：

```text
start-school-pilot.cmd
```

适合日常值班老师、管理员直接使用。

在项目目录 `D:\xiaowuxitong` 下执行：

```powershell

```

如果只是当前会话临时启动，保持窗口不要关闭。

看到以下地址能够打开，就说明服务已启动：

```text
http://<current LAN IP>:3000/login
```

这时应该重新运行一次 `start-school-pilot.cmd`，确保正在监听 `3000` 的进程和最新构建一致。

## 六、各角色详细使用说明

## 1. 系统管理员 `admin`

适合学校最高权限管理员使用。

### 主要功能

- 进入用户管理页面：`/dashboard/users`
- 新增账号
- 修改账号角色

- 维护年级、班级、部门、学科

## 九、备份与恢复的最简说明

当前数据库运维建议只由管理员或网管执行。

系统管理员在 `/dashboard/data-management` 执行单条删除、批量删除或一键清理时，系统会先调用 `pg_dump` 自动生成 PostgreSQL custom dump。备份失败时删除会被取消；备份成功和删除结果都会写入审计日志。默认备份目录是：

```text
D:\xiaowuxitong\artifacts\data-management-backups
```

如需调整 `pg_dump` 路径或备份目录，可在环境变量中配置 `PG_DUMP_PATH` 和 `DATA_BACKUP_DIR`。

### 1. 备份

当前已经演练通过的思路是：

- 使用 `pg_dump`
- 备份 `school_affairs`
- 生成 `.dump` 文件

当前已验证的备份样例文件：

```text
D:\xiaowuxitong\artifacts\backup-smoke\school_affairs_20260424_200426.dump
```

### 2. 恢复

恢复时建议：

1. 先创建临时恢复库
2. 用 `pg_restore` 恢复进去
3. 检查用户数、学生数、教师数、检查记录数
4. 确认无误后再做正式恢复操作

## 十、常见问题

## 1. 登录页打不开

先检查：

- 这台试点电脑是否已经启动 `start-school-pilot.cmd`
- 当前访问地址是不是 `http://<current LAN IP>:3000/login`
- 试点电脑是否仍在学校同一网络里

如果浏览器显示的不是普通空白页或网络错误，而是：

```text
This page couldn’t load
```

优先按下面顺序检查：

- 这台电脑是否刚执行过 `npm run build`，但没有重启试点站点
- 是否仍是旧的 `next start` 进程在占用 `3000`
- 重跑一次 `start-school-pilot.cmd`

这类情况往往不是账号问题，而是前端静态资源版本和正在运行的站点不一致。常见表现是某个 `/_next/static/chunks/*.js` 资源加载失败，并出现 `ChunkLoadError`。

如果 `/login` 能打开，账号也能登录，但 `用户权限`、`数据管理中心`、`师生档案`、`往届存档`、`常规检查`、`统计导出` 这类数据页面显示同样的 server error，则优先联系技术维护人员检查 PostgreSQL 表结构是否已经同步到当前代码。典型原因是代码已经更新到新 Prisma schema，而 live 数据库还缺少新列。维护人员应在项目根目录执行 `npm run db:push`，必要时按 Prisma 提示确认新增约束，然后重新打开这些业务页验证。

## 2. 别的办公电脑打不开，但本机能打开

优先检查：

- 试点站点是否正在运行
- 试点电脑 IP 是否变化
- Windows 防火墙是否拦截 `3000` 端口

## 3. 能打开登录页，但账号登录失败

优先检查：

- 用户名是否输对
- 密码是否输对
- 账号是否被系统管理员停用
- 是否刚被系统管理员重置过密码

## 4. 为什么 `admin` 还能登录

当前 `admin / ChangeMe123!` 既是数据库里的系统管理员账号，也是 `.env.local` 里的兜底管理员配置。

这意味着：

- 数据库正常时，它可以作为系统管理员登录
- 即使数据库暂时不可用，只要环境变量没改，它仍可能作为兜底管理员登录

如果后面学校要更严谨地长期使用，建议：

- 修改数据库里的 `admin` 密码
- 同时修改 `.env.local` 里的 `BOOTSTRAP_ADMIN_PASSWORD`
- 或者进一步收紧兜底管理员的使用方式

## 十一、当前最重要的管理建议

- 不要把数据库运维账号发给普通老师
- 普通老师只需要网页登录账号
- 试点稳定后，建议第一时间改掉默认密码 `ChangeMe123!`
- 建议系统管理员把账号清单单独保存，并记录谁在使用哪一个账号
- 建议每天至少备份一次数据库
- 建议先让高二指定老师试用，再根据反馈调整
