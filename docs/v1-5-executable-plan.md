# V1.5 当前可执行计划

## 定位

这份计划是 V1 试点后的近期执行建议。2026-05-11 已先落地 V1.5 Step 8 申请审批模块；本文后续冲刺建议主要保留给数据质量、导入预检、常用筛选导出、学生一页通和轻量考勤/请假方案。

默认执行窗口为 `2 周冲刺`，目标是优先解决真实学校使用中最容易影响效率和数据可信度的问题。

## 本轮目标

| 目标 | 成功标志 |
| --- | --- |
| 降低真实数据导入风险 | 上传前能预检，错误行能定位和修复，避免静默污染数据 |
| 减少重复筛选导出 | 常用筛选和字段选择可以复用，领导和年级办公室不用每次重配 |
| 提高学生查询效率 | 学生一页通能集中查看基础档案、状态和近期管理记录 |
| 控制范围膨胀 | 考勤/请假先完成轻量方案，不直接扩成完整教务系统 |
| 强化真实试点安全 | 导出、敏感字段、备份、登录兜底策略有明确检查项 |

## 优先级拆解

### P0 数据质量与导入预检

建议先做，因为它直接影响真实数据能不能放心进系统。

需要实现的能力：

- 预检上传文件，不立即写入数据库。
- 展示本次导入会新增多少、更新多少、失败多少、警告多少。
- 错误定位到行号、字段、原因。
- 支持下载错误行，方便操作人员在 Excel 中修正。
- 规则至少覆盖身份证号为空或重复、姓名为空、年级/班级不存在、动态字段列名不匹配、学生归档范围错误。

验收场景：

- 同一身份证号再次导入时显示为更新，而不是新增重复记录。
- 班级不属于所选年级时阻止写入。
- 动态字段删除后，旧模板中的列能给出清晰提示。
- 没有导入权限的账号不能触发预检或提交写入。

### P1 常用筛选、字段显示和导出模板

建议第二个做，因为它提升日常效率，且可复用现有筛选和导出逻辑。

需要实现的能力：

- 保存常用筛选条件，例如“高二在校学生名单”“某部门教师名单”“本月学生量化汇总”。
- 允许选择导出字段，默认仍使用当前完整安全字段集。
- 模板至少支持个人级；若学校明确需要，再扩展到角色共享或全校共享。
- 导出模板继续走现有权限和审计日志。

验收场景：

- 年级管理员保存的模板不能越权查询其他年级。
- 删除或停用动态字段后，模板能提示字段已变化，而不是导出错列。
- 校领导能复用常用导出，不需要每次重新配置筛选。

### P1 学生一页通

建议第三个做，因为它把现有数据整合成更适合管理者使用的入口。

首版页面建议包含：

- 学生基础信息、年级、班级、状态、宿舍相关字段。
- 监护人或联系电话字段，如果当前学校已配置。
- 近 30 天或本月常规检查摘要。
- 最近一次状态变更或档案更新时间。
- 快捷返回来源页面和权限内导出入口。

验收场景：

- 从 `/dashboard/quick/students` 能进入学生一页通。
- 从师生档案筛选结果能进入学生一页通。
- 年级管理员只能打开自己绑定年级的学生详情。
- 归档学生默认不混入在校学生详情入口。

### P2 轻量考勤/请假方案

本轮建议先完成方案冻结，不建议直接扩成完整教务系统。

需要确认的问题：

- 学校是按天、半天还是课节记录考勤。
- 请假是否需要审批流，还是只记录状态。
- 考勤统计是否要进入领导导出中心。
- 考勤和现有常规检查中的“纪律/出勤检查”如何区分。

最小可行版本建议：

- 班级维度登记日常出勤。
- 学生状态包含正常、请假、迟到、早退、缺勤。
- 支持按日期、年级、班级导出。
- 不做复杂审批和家长端通知。

### P2 通知、待办与安全运维

这部分可以作为冲刺后半段或下一轮任务。

建议优先项：

- 导入失败待处理提醒。
- 备份失败或长时间未备份提醒。
- 导出原因填写和审计记录展示。
- 手机号、身份证等敏感字段的列表页脱敏展示。
- 生产环境 Bootstrap fallback 收紧或关闭方案。

## 不做事项

本轮不做以下内容：

- 课程表、排课、调课。
- 考试成绩和成绩分析。
- 家长端或学生端。
- 财务缴费、食堂、资产。
- 多校 SaaS。
- 重型 AI 分析。

- 已确认真实试点数据范围。
- 已确认 V1.5 首批功能是否包含轻量考勤开发，还是只做方案。
- 已确认必填字段和敏感字段口径。
- 已确认导入失败时是允许部分成功，还是整表必须全部通过。

- 已更新 `memory-bank/prd.md`、`memory-bank/implementation-plan.md`、`memory-bank/progress.md` 和 `memory-bank/architecture.md`。

## 推荐下一步

如果只批准一个 2 周冲刺，建议执行顺序为：

1. 数据质量与导入预检。
2. 常用筛选、字段显示和导出模板。
3. 学生一页通。
4. 轻量考勤/请假方案冻结。

这样可以最大化复用现有师生档案、统计导出、权限和审计能力，同时避免过早进入课表、成绩、门户等大模块。
