# 数据安全与部署计划

## 基本原则

- 真实师生数据只放在受控服务器环境中。
- 当前正式运行环境是腾讯云轻量应用服务器，不再使用 Windows 试点电脑、本地 PostgreSQL 或本地加隧道作为主方案。
- PostgreSQL 只监听服务器本机，不开放公网数据库端口。
- 权限边界必须在服务端执行，不能只依赖页面隐藏按钮。
- 账号、导出、误录删除、申请审批和敏感变更要留下审计记录。
- 发布必须经过备份、schema 同步、构建、重启和 smoke。

## 当前部署边界

- 公网入口：`http://124.222.136.121/login`
- 应用服务器：腾讯云轻量应用服务器 Ubuntu 22.04
- 应用目录：`/opt/xiaowuxitong/app`
- 数据库：服务器本机 PostgreSQL，数据库 `school_affairs`
- 备份：`/opt/xiaowuxitong/backups`
- 服务：`xiaowuxitong.service`
- 反向代理：Nginx `:80`

公网安全组只应开放必要端口：

- `22`：SSH 管理
- `80`：当前 Web 入口
- `443`：域名备案和证书完成后启用 HTTPS

不开放：

- PostgreSQL `5432`
- Next.js 内部端口 `3000`

## 数据策略

- 当前云端 PostgreSQL 不保留旧演示测试数据。
- 本地开发数据库不向云端迁移。
- `db:seed:demo`、`db:clear:demo`、PGlite 模拟只用于本地开发或一次性验证。
- 正式数据通过页面导入、表单维护或受控迁移进入云端数据库。
- 大批量导入前必须先备份。

## 角色与权限模型

当前系统支持：

- 系统管理员：用户、权限、学校结构、数据管理和全量业务能力。
- 校领导：查看全校师生、检查、统计和申请记录。
- 部门领导：按职责处理部门相关审批和管理工作。
- 年级管理员：绑定一个负责年级，只能访问该年级范围数据。
- 政教工作人员：检查分类、项目、记录、误录纠错和统计导出。
- 教务工作人员：教师与学生档案维护、导入和导出。
- 行政办公人员：学校行政办公类申请处理。
- 后勤办公人员：日常报修类申请审批。
- 一线教师：提交并跟踪本人申请。
- 学生账号：当前仅用于自助密码维护。

已落实的安全边界包括：

- 数据库账号登录与密码哈希存储
- 账号启用、停用与管理员重置密码
- 浏览器会话边界
- 年级管理员服务端范围限制
- 导出审计日志
- 用户管理审计日志
- 申请提交、审批通过、审批驳回审计日志
- 审批职责服务端范围校验
- 数据管理中心硬删除前备份保护

## 发布与备份策略

日常发布使用：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\deploy-tencent-lighthouse.ps1
```

服务器脚本会先执行：

```bash
bash scripts/server/backup-postgres.sh
```

备份默认保留最近 10 份。正式使用后建议：

- 定期把备份复制到腾讯云 COS 或其他异地存储。
- 每月至少做一次恢复演练。
- 为大批量导入、权限调整和 schema 变更保留额外手动备份。

## 上线前检查

- [ ] 公网登录页可打开。
- [ ] PostgreSQL 不对公网开放。
- [ ] `.env.local` 权限受控且不进 Git。
- [ ] Bootstrap 管理员和数据库密码已安全保存。
- [ ] Prisma schema 已同步到云端 PostgreSQL。
- [ ] authenticated smoke 已通过。
- [ ] 正式账号已创建，临时密码已要求修改。
- [ ] 审批职责按学校真实岗位配置。
- [ ] 数据导入前已完成手动备份。

## 当前限制

- 当前云端库为空库运行口径，不带旧演示数据。
- 域名和 HTTPS 需要备案与证书配置完成后再切换。
- 40GB 系统盘空间有限，需要持续关注备份占用。
- 自动部署目前使用 `prisma db push`，正式长期运维可再评估受控 migration 流程。

## 建议下一步

1. 确认正式访问域名和备案计划。
2. 配置 HTTPS。
3. 设定备份异地保存策略。
4. 按学校真实岗位创建账号和审批职责。
5. 在导入真实数据前完成一次手动备份和恢复演练。
