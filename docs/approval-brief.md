# 校务系统试点审批说明

## 申请事项

申请在当前学校试点电脑上启用 PostgreSQL 数据库环境，用于校务系统高二真实环境试点。

## 背景问题

学校目前大量校务统计依赖 Excel 文件：

- 师生信息在不同部门之间容易出现版本不一致
- 常规检查结果需要重复录入和手动汇总
- 临时统计和上报需要人工合并多个表格
- 领导需要数据时，响应速度受制于人工整理
- 缺少统一权限和审计记录

## 当前成果

系统已经完成以下可演示能力：

- 登录和用户权限管理
- 年级、班级、部门、学科配置
- 教师和学生档案维护、误录修改与删除
- Excel 模板导入
- 按条件筛选和导出
- 常规检查项目配置
- 常规检查结果录入、误录修改与删除
- 检查统计汇总
- Excel/CSV 报表导出
- 本地模拟数据库和虚构演示数据

## 为什么需要 PostgreSQL

PostgreSQL 是系统的数据存储基础，用来保存：

- 用户账号和权限
- 学校结构
- 教师档案
- 学生档案
- 常规检查记录
- 导出审计记录

当前本机模拟数据库可以证明功能流程，但不能替代真实 PostgreSQL 试点环境。只有在当前试点电脑上启用 PostgreSQL 后，才能进行真实浏览器烟测、小范围试点和备份恢复演练。

## 建议试点方式

1. 先在当前试点电脑上启用 PostgreSQL，并把数据目录放在 `D` 盘。
2. 写入虚构演示数据完成真实环境烟测。
3. 先选择高二年级做小范围试点。
4. 试点期间保留现有 Excel 流程。
5. 试点通过后再决定是否扩大范围。

## 风险控制

- 不直接使用真实全校数据启动。
- 不替代现有 Excel 流程，先并行试点。
- 系统账号按角色授权。
- 导出和敏感操作记录审计日志。
- PostgreSQL 和 Web 应用优先放在当前试点电脑，办公电脑只访问系统页面。
- 正式使用前完成备份和恢复演练。

## 需要领导决定

- 是否同意在当前试点电脑上启用 PostgreSQL。
- 是否同意安排一次真实环境浏览器烟测。
- 是否同意先按高二年级范围开展首轮试点。

## 建议结论

建议批准在当前试点电脑上启用 PostgreSQL 试点环境。  
这一步投入较小、风险可控，可以帮助学校判断系统是否值得进入真实试点。
